SQL Injection Attack

SQL code injection technique involves placement of malicious code in SQL statements via web page input which may disrupt/destroy SQL database. For example: ‘ OR ‘1’=1’ -- submitted into an input field would cause this parameter to be true for a statement such as : SELECT * FROM login WHERE user = + “userParam”. Since ‘1’=’1’ this statement returns true thus causing SQL to select all counts of login. In order to prevent this attack, one must accept parameters as: SELECT * FROM login WHERE user = ?. Then one must setString(1, userParam) which causes the input to be placed directly in the database where the sql string will not get manipulated.
